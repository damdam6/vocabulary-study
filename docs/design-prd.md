### 1.2 타이포그래피

- **UI 폰트**: `'Noto Sans KR', -apple-system, 'Apple SD Gothic Neo', sans-serif`
- **한자 폰트**: `'Noto Sans SC', 'Noto Sans KR', sans-serif` — **Sans 고정** (serif 옵션 없음). 한자 표시 요소는 항상 `lang="zh-Hans"`.
- **웹폰트 로딩**: Google Fonts CDN (`Noto Sans KR` 400/500/700 + `Noto Sans SC` 400/500/700), `display=swap`, `preconnect` 2건. CDN 실패 시 시스템 폰트 폴백으로 동작에 지장 없어야 한다.
- 한자 카드 글자 크기(글자 수 적응): **2자 이하 84px / 3자 64px / 4자 이상 52px**. 카드 뒷면·결과 화면 한자 44~50px, 모드2 입력란 28px.
- 숫자는 `font-variant-numeric: tabular-nums` (현황 카드, 진행도, 완료 화면).

### 1.3 형태·모션

- radius: 주 버튼 14~16px, 문제 카드 22px, 현황 카드 15px, 칩 999px.
- 문제 카드 그림자: `0 20px 40px rgba(0,0,0,0.35)`.
- 버튼 press: `transform: scale(0.98)` (원형 O/X는 0.94).
- 애니메이션:
  - `flip` — 모드1 카드 3D 플립, `rotateX`, 0.55s `cubic-bezier(0.3,0.7,0.2,1)`
  - `popfb` — O/X 대형 글리프(130px) 팝인 후 페이드아웃, **총 0.4s** (목업 0.68s에서 단축, §5.1 참조)
  - `fadeup` — 요소 등장, 0.3s
  - `shake` — 오답·비밀번호 오류, 0.4s

### 1.4 레이아웃

- 모바일 우선. 앱 컬럼: `width:100%; max-width:430px` 중앙 정렬, `min-height:100dvh`, 좌우 1px `border-faint`. 데스크톱에서도 동일(중앙 컬럼).
- 기준 뷰포트 390×844.

## 2. 화면: 로그인

- 구성: 로고 → 타이틀 "단어 암기" → 안내 2줄("구글 시트의 단어로 오늘의 퀴즈를 만듭니다. / 비밀번호를 입력하세요.") → 비밀번호 입력(54px, 자동 포커스) → "들어가기" 버튼(54px).
- 목업의 각주("프로토타입 — 아무 값이나…")는 **삭제**.
- **검증 방식**: 제출 시 입력값을 Bearer로 `GET /api/health` 호출. `200` → localStorage 저장 후 홈으로. `401` → 오류 상태.
- **검증 중**: 버튼 비활성 + 라벨 "확인 중…".
- **오류 상태**: 입력란 `shake` + 테두리 `red`, 입력란 아래 13px `red` 텍스트 "비밀번호가 올바르지 않습니다". 재입력 시작하면 오류 표시 제거.
- 세션 중 어느 API든 `401` 수신 시: 저장값 삭제 후 이 화면으로 복귀 (기능 PRD §8).

## 3. 화면: 홈

- 구성(위→아래): 날짜(`2026년 7월 17일 금요일` 형식, **Asia/Seoul 기준**) → 타이틀 "오늘의 학습" → 현황 카드 3개 → (스페이서) → 미전송 인디케이터(조건부) → "오늘 세션 · n문제" → "학습 시작" 버튼(58px).
- 현황 카드: **복습 대기**(`accent-light`) / **학습 중**(`text`) / **졸업**(`green`). 졸업 수는 복습 대기를 **포함한** 졸업 단어 총수 (`reviewDue + reviewScheduled`).
- **"오늘 세션 · n문제"**: 큐 구성 로직(기능 PRD §6.1)과 동일한 계산의 총 문제 수. `min(60, min(복습대기, 60) + 학습중 단어 수)` — 학습 중 단어는 단어당 1문제 (#44).
- 시작 버튼: n=0이면 비활성(`bg-disabled`/`text-faint`) + 라벨 "오늘 할 것 없음".
- **로딩 상태**: 현황 카드 3개 자리에 같은 크기의 스켈레톤(펄스), 버튼 비활성 + "불러오는 중…". 세션 수 줄은 숨김.
- **에러 상태**: 카드 영역 대신 에러 카드 — "단어를 불러오지 못했습니다" + 원인 한 줄(muted) + "다시 시도" 버튼(`accent`).
- **미전송 인디케이터**: 재시도 큐(이슈 #18)에 항목이 있을 때만, 시작 버튼 위에 한 줄 — `● 미전송 기록 n건 · 연결되면 자동 저장` (13px, 점만 `red`, 텍스트 `text-muted`). 홈에서만 표시하고 학습 중에는 아무것도 띄우지 않는다.
- 홈 진입 시마다(로그인 직후·세션 종료 복귀 포함) `/api/words` 재조회.

## 4. 화면: 학습

### 4.1 공통 프레임

- 헤더: 좌 "종료" 보조 버튼(즉시 홈 복귀, 확인 다이얼로그 없음) / 중앙 진행도 `현재 / 전체`.
- **진행도 분모는 동적**: 학습 중 오답 재삽입으로 큐가 늘면 분모도 즉시 증가.
- 헤더 아래 중앙 칩: 복습 문제면 "복습" 뱃지(파랑 틴트) + 모드 칩("한자 → 뜻" / "뜻 → 한자").
- 날짜·"오늘" 판정은 반드시 `getSeoulToday()`(`src/lib/wordState.ts`) 사용 — 목업의 기기 로컬 시간 로직을 그대로 옮기지 말 것.

### 4.2 모드 1 (한자 → 뜻, 자가판정)

- 앞면: 한자(§1.2 크기 스케일) + 하단 힌트 "탭해서 뜻 보기". 하단 영역 힌트 "카드를 탭하면 뜻이 보입니다"(`text-hint-dim`).
- 카드 탭 → 3D 플립 → 뒷면: 한자 44px + 병음(`accent-light` 20px) + 뜻 23px.
- 공개 후 하단에 O/X 원형 버튼(82px) `fadeup` 등장 — **X 왼쪽("몰랐음") / O 오른쪽("알고 있었음")**.
- **O/X 탭 → 판정 즉시 기록 + 즉시 다음 문제로 전환.** 대기 시간 없음.

### 4.3 모드 2 (뜻 → 한자, 자동 채점)

- 출제: 뜻 카드(27px) + "이 뜻의 한자를 입력하세요" + 입력란(60px, 중앙 정렬 28px, 자동 포커스, `autocomplete/autocorrect/autocapitalize off`, `spellcheck=false`, `enterKeyHint="done"`, `lang="zh-Hans"`, placeholder "汉字") + "제출" 버튼. Enter 제출 가능.
- 채점: 트림 후 정확 일치. 빈 입력 = 오답 (기능 PRD §5.2).
- **정답 → 결과 화면 없이 O 글리프 피드백 + 즉시 다음 문제.**
- **오답 → 결과 화면**: "오답"(`red`, shake) + 정답 한자 50px + 병음 + 뜻 + (입력했었다면) "내 답: ~~입력값~~"(취소선 `red`) → **"다음" 버튼으로만 진행**.

### 4.4 정오 피드백 오버레이

- O/X 대형 글리프(130px)가 화면 중앙에 `popfb`로 떴다 사라짐 + 배경 틴트. `pointer-events: none`.
- **논블로킹**: 다음 문제 카드가 이미 전환된 위에 겹쳐 재생된다. 진행을 막거나 지연시키지 않는다.
- 발생 시점: 모드1 O/X 탭 직후, 모드2 정답 제출 직후. (모드2 오답은 결과 화면이 피드백이므로 오버레이 없음)

### 4.5 진행 규칙 (기능 PRD §6.2 재확인)

- 학습 중 오답: API 호출 없음, 해당 문제를 큐 맨 뒤에 1회 재삽입(60문제 상한 내).
- 복습 오답: `POST /api/review-fail`, 재출제 없음.
- 기록 전송은 비동기 — 실패해도 진행을 막지 않고 재시도 큐(#18)로.

## 5. 화면: 세션 완료

- 중앙: ✓ 원형(76px, green 틴트) → "세션 완료" → "기록은 문제 단위로 이미 저장되었습니다"(muted) → 정답 수(`green` 40px) | 구분선 | 오답 수(`red` 40px) → 하단 "홈으로" 버튼.
- 정답/오답 수는 이번 세션의 모드1·2 합산 (X 자가판정도 오답으로 집계).

## 8. 결정 로그

| 항목 | 결정 | 기각한 대안 |
|---|---|---|
| 웹폰트 | Google Fonts CDN + swap + 폴백 | 셀프 호스팅 서브셋 (시트에 단어가 계속 추가돼 정적 서브셋 위험) |

| 모드1 진행 | 즉시 자동 진행 | 딜레이 후 진행(목업), "다음" 버튼, 화면 탭 진행 |
| 모드2 정답 | 즉시 자동 진행 | 결과 화면 + "다음"(목업·구 PRD) |
| 진행도 분모 | 동적 증가 | `n/60` 고정 |
| 홈 세션 수 | "오늘 세션 · n문제" 표시 | 미표시 |
| 미전송 표시 | 홈에만 한 줄 인디케이터 | 무표시, 토스트 |
