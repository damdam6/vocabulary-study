# 중국어 단어 암기 사이트 PRD

> 이 문서는 구현에 필요한 모든 결정사항을 담은 단일 참조 문서다.
> 컨텍스트 없이 이 문서만 읽고 v1을 구현할 수 있어야 한다.
>
> 작성일: 2026-07-13 · 상태: 확정 (미결 사항은 §12에 표시) · 2026-07-19 개정: 단어 등록 v1 편입 (#50)

## 1. 개요

**한 줄 요약**: 구글 시트로 단어를 관리하고, 웹에서 2가지 퀴즈로 암기한 뒤, 고정 간격 반복으로 장기 기억에 넣는 개인용 중국어 단어 암기 사이트.

- **사용자**: 소유자 1인 전용. 회원가입·멀티유저 없음.
- **단어 원천 데이터(source of truth)**: 구글 시트. 단어 등록은 claude.ai 추출 채팅(프로젝트 킷, `docs/registration-kit/`)으로 스키마 JSON을 만들어 사이트의 단어 등록 화면에 붙여넣는 방식이다 (§7.3, §9.3). 자격증명(서비스 계정 키)은 Worker에만 존재하고 기기·프롬프트에 배포하지 않는다. 상세 사양 원본: `docs/plans/word-registration-system.md`. *(2026-07-19 단어 등록 v1 편입으로 변경)*
- **사이트의 역할**: 시트를 읽어 퀴즈를 출제하고, 학습 결과(정답 수, 다음 복습일, 정답 타임스탬프)를 시트에 다시 쓰며, 등록 화면 경유로 새 단어를 시트에 추가한다.

## 2. 사용 시나리오

1. 사용자가 수업/교재에서 만난 단어(캡처·깨진 텍스트)를 claude.ai 추출 채팅으로 스키마 JSON으로 뽑아 복사한다 (리포 밖, 프로젝트 킷 사용). *(2026-07-19 단어 등록 v1 편입으로 변경)*
2. 사이트의 단어 등록 화면에 붙여넣고, 검증 통과 후 탭을 선택해 시트에 등록한다 (§9.3).
3. 폰(주 환경) 또는 PC에서 사이트에 접속한다. 첫 접속 시 비밀번호 1회 입력.
4. 홈에서 오늘의 현황(복습 대기 n개 / 학습 중 n개 / 졸업 n개)을 보고 "학습 시작"을 누른다.
5. 최대 60문제 세션을 진행한다. 앱이 모드 1·2를 자동으로 섞어 출제한다.
6. 정답을 낼 때마다 결과가 시트에 즉시 기록된다.
7. 단어는 모드별 3회 정답을 달성하면 "졸업"하고, 이후 1→3→7→14→30일 고정 간격 복습으로 전환된다.

## 4. 데이터: 구글 시트 계약

이 절은 **시트에 쓰는 모든 주체가 지켜야 하는 계약**이다. v1에서 시트 쓰기는 사이트(Worker)로 일원화된다 — 학습 기록(D열 이후)과 등록 화면 경유 신규 행(A~C열, append-only) 모두. *(2026-07-19 단어 등록 v1 편입으로 갱신 — 계약 자체는 불변)*

### 4.1 스프레드시트 구조
- 스프레드시트 1개. ID는 Worker 환경변수 `SHEET_ID`로 주입.
- **탭(시트) 여러 개** = 단어 출처별 분류 (예: `HSK4`, `회화수업`, `교재5과`). 탭 이름은 자유.
- 모든 탭이 동일한 컬럼 계약을 따르며, 앱은 **모든 탭을 통합**해서 하나의 학습 큐로 다룬다.
- 탭 이름이 `_`(언더스코어)로 시작하면 학습 대상에서 제외한다 (메모·설정용 탭 escape hatch).
- 각 탭의 1행은 헤더 행으로 간주하고 건너뛴다.

### 4.2 컬럼 계약 (모든 단어 탭 공통)

| 열 | 내용 | 형식 | 쓰는 주체 |
|---|---|---|---|
| A (1열) | 한자 (간체) | 텍스트, 탭 내 유일 키 | 사이트 (등록 화면, §7.3) |
| B (2열) | 병음 | 텍스트 | 사이트 (등록 화면, §7.3) |
| C (3열) | 한국어 뜻 | 텍스트 | 사이트 (등록 화면, §7.3) |
| D (4열) | 모드 1 정답 수 | 정수, 신규 등록 시 빈칸 또는 0 | 사이트 |
| E (5열) | 모드 2 정답 수 | 정수, 신규 등록 시 빈칸 또는 0 | 사이트 |
| F (6열) | 다음 복습일 \| 현재 간격 | `YYYY-MM-DD\|일수` (예: `2026-08-12\|14`). 미졸업 단어는 빈칸 | 사이트 |
| G (7열)~ | 정답 타임스탬프 기록 | 정답 1회마다 다음 빈 셀에 1개 append. 형식: `YYYY-MM-DD HH:mm\|m1` 또는 `\|m2` (Asia/Seoul 기준) | 사이트 |

주의사항:
- **정답만 기록한다.** 오답은 어디에도 기록하지 않는다 (카운트·타임스탬프 모두).
- F열에 간격(일수)을 함께 저장하는 이유: 복습 오답 시 "간격 한 단계 후퇴"를 구현하려면 현재 간격을 알아야 하는데, 다음 복습일만으로는 오답 이후 상태에서 복원이 불가능하다.
- 행 식별: 사용자가 시트를 정렬/삽입할 수 있으므로 행 번호를 캐시하지 않는다. 쓰기 시점에 Worker가 **탭 이름 + A열 한자**로 행을 다시 찾아 쓴다. 같은 한자가 여러 탭에 있으면 각 탭의 행은 독립된 단어로 취급한다 (진도도 따로).

### 4.3 예시

```
탭 "HSK4"
| A    | B      | C        | D | E | F              | G                  | H                  | ...
| 经济 | jīngjì | 경제     | 3 | 3 | 2026-07-20|7   | 2026-07-01 09:12|m1 | 2026-07-01 09:15|m2 | ...
| 严重 | yánzhòng | 심각하다 | 2 | 0 |                | 2026-07-10 21:02|m1 | 2026-07-11 08:40|m1 | ...
```

## 5. 학습 모델

### 5.1 단어 상태 머신

상태는 별도 컬럼 없이 D·E·F열에서 파생한다.

| 상태 | 판정 조건 | 의미 |
|---|---|---|
| **학습 중** | D < 3 또는 E < 3 | 모드별 3회 정답을 채우는 중 (신규 포함) |
| **복습 대기** | D ≥ 3 그리고 E ≥ 3 그리고 F의 날짜 ≤ 오늘 | 오늘 복습해야 함 |
| **복습 예약** | D ≥ 3 그리고 E ≥ 3 그리고 F의 날짜 > 오늘 | 아직 복습일 안 됨, 출제 안 함 |

- "졸업" = D ≥ 3 && E ≥ 3 최초 달성. 졸업 순간 F열에 `내일 날짜|1`을 쓴다.
- 오답이 나와도 **어떤 카운트도 리셋되지 않는다.** 오답의 효과는 (a) 학습 중: 없음(카운트 정지), (b) 복습: 간격 후퇴(§5.3)뿐이다.

### 5.2 퀴즈 모드

**모드 1 — 한자 보고 뜻 맞추기 (자가 판정)**
1. 한자만 크게 표시.
2. 사용자가 화면(또는 "뜻 보기" 버튼)을 탭 → 병음 + 한국어 뜻 공개.
3. `O`(알고 있었음) / `X`(몰랐음) 버튼으로 자가 판정.
4. O → 정답 처리(D열 +1, 타임스탬프 기록). X → 아무 기록 없음.
5. 자동으로 다음 문제로 진행.

**모드 2 — 뜻 보고 한자 입력 (자동 채점)**
1. 한국어 뜻만 표시.
2. 사용자가 텍스트 입력란에 한자 입력 (OS 중국어 IME 사용 전제). "제출" 버튼 또는 Enter.
3. 채점: 입력값 앞뒤 공백 제거 후 A열 한자와 **정확 일치**해야 정답. (병음 입력, 부분 일치, 이체자 허용 없음)
4. 결과 화면: 정답/오답 표시 + 정답 한자·병음·뜻 함께 표시.
5. 정답 → E열 +1, 타임스탬프 기록. 오답 → 기록 없음.
6. 정답 시 자동으로 다음 문제로 진행. 오답 시에만 결과 화면 + "다음" 버튼으로 진행 (정답을 볼 시간 확보). *(2026-07-17 디자인 확정으로 변경, `docs/design-prd.md` §4.3)*

**빈 입력으로 제출 시**: 오답으로 처리 (모른다는 뜻).

### 5.3 고정 간격 반복 (졸업 후)

- 간격 사다리: **1일 → 3일 → 7일 → 14일 → 30일**. 30일 도달 후에는 계속 30일 간격 유지 (완전 제거되는 "마스터" 상태 없음).
- 복습 문제는 **모드 1·2 중 무작위**로 출제한다.
- **복습 정답**: 해당 모드 카운트 +1, 타임스탬프 기록, 간격 한 단계 상승, F열 = `오늘+새 간격|새 간격`.
- **복습 오답**: 기록 없음. 간격 한 단계 후퇴(최소 1일), F열 = `오늘+후퇴한 간격|후퇴한 간격`. 그 단어는 이번 세션에서 다시 출제하지 않는다.
- 복습일을 지나쳐도 페널티 없음. 만기된 복습은 계속 "복습 대기"로 쌓일 뿐이다.

### 5.4 시간 기준

- 모든 날짜·시간은 **Asia/Seoul** 기준. "오늘"의 경계는 서울 자정.
- 타임스탬프 형식: `YYYY-MM-DD HH:mm` (초 불필요).

## 6. 세션 규칙

### 6.1 큐 구성

세션 시작 시 전체 탭을 통합해 아래 순서로 큐를 만든다. **상한 60문제.**

1. **복습 대기 단어** 전부 (단어당 1문제, 모드는 무작위). 60개를 넘으면 복습일이 오래된 것부터 60개.
2. 남은 슬롯에 **학습 중 단어**: 단어당 1문제 —
   - D < 3만 해당하면 모드 1, E < 3만 해당하면 모드 2,
   - 둘 다 해당하면 모드 하나를 무작위 선택 (복습 문제의 모드 무작위와 동일한 방식).
   - 정렬: 총 정답 수(D+E)가 많은 단어 우선 (거의 다 된 단어부터 졸업시킴), 동률이면 시트 상 순서.
3. 1~2를 합친 뒤 전체를 셔플한다. 단어당 1문제이므로 같은 단어가 세션 큐에 두 번 들어가지 않는다 (§6.2 오답 재삽입 제외).

### 6.2 진행 규칙

- 세션 중 오답이 난 **학습 중** 단어의 해당 문제는 큐 뒤쪽에 1회 재삽입한다 (60문제 상한 내에서). 복습 단어의 오답은 재삽입하지 않는다 (§5.3).
- 사용자는 언제든 세션을 중단할 수 있다. 기록은 문제 단위로 이미 시트에 반영돼 있으므로 유실 없음. "세션 완료" 상태를 따로 저장하지 않는다.
- 큐가 소진되면 완료 화면(이번 세션 정답/오답 수)을 보여주고 홈으로.

## 7. 시스템 아키텍처

```
[브라우저: React SPA]
   │  fetch /api/* (Authorization: Bearer <비밀번호>)
   ▼
[Cloudflare Worker]  ← 정적 자산(Workers Assets)도 같은 Worker가 서빙
   │  Google Sheets API v4 (서비스 계정 JWT)
   ▼
[구글 스프레드시트]
```

### 7.1 스택

- **프론트엔드**: React + Vite. SPA 1개, 화면은 홈/학습/단어 등록 3개뿐이므로 상태 기반 화면 전환으로 충분 (라우터 라이브러리 선택 자유). 등록 화면의 병음-한자 일치 검증에 `pinyin-pro` 사전 라이브러리를 사용하되, 등록 화면 전용 청크로 지연 로드(dynamic import)해 홈·학습 초기 번들 크기에는 영향을 주지 않는다. *(2026-07-19 단어 등록 v1 편입으로 변경)*
- **백엔드**: Cloudflare Worker 1개. 정적 자산 서빙(Workers Assets) + `/api/*` 처리를 겸한다. 별도 서버 없음.
- **시트 접근**: Google 서비스 계정. 서비스 계정 이메일을 스프레드시트에 **편집자**로 공유해 둔다. Worker가 서비스 계정 키(JSON)로 JWT를 만들어 Sheets API v4 호출. 액세스 토큰은 Worker 메모리에 캐시(만료 ~1시간).
- **읽기/쓰기 모두 Worker 경유.** 시트는 웹 게시하지 않고 비공개 유지한다.

### 7.2 환경변수 / 시크릿 (Worker)

| 이름 | 내용 |
|---|---|
| `SHEET_ID` | 스프레드시트 ID |
| `GOOGLE_SERVICE_ACCOUNT_KEY` | 서비스 계정 키 JSON 전체 (secret) |
| `APP_PASSWORD` | 접근 비밀번호 (secret) |

### 7.3 API 명세

모든 요청에 `Authorization: Bearer <APP_PASSWORD>` 필수. 불일치 시 `401`.

**`GET /api/words`**
- 모든 학습 대상 탭(이름이 `_`로 시작하지 않는 탭)의 전 단어를 반환.
- 응답 예:
```json
{
  "fetchedAt": "2026-07-13 09:00",
  "words": [
    {
      "tab": "HSK4", "hanzi": "经济", "pinyin": "jīngjì", "meaning": "경제",
      "m1": 3, "m2": 3, "nextReview": "2026-07-20", "interval": 7
    }
  ]
}
```
- D·E열이 빈칸이면 0으로, F열이 빈칸이면 `nextReview: null, interval: null`로 정규화한다.
- 큐 구성(§6.1)과 상태 판정(§5.1)은 **클라이언트**가 수행한다. Worker는 데이터 중계만.

**`POST /api/answer`** — 정답 1건 기록 (오답은 호출하지 않음)
- 요청:
```json
{ "tab": "HSK4", "hanzi": "经济", "mode": "m1", "timestamp": "2026-07-13 09:12", "isReview": true }
```
- Worker 동작 (한 요청 안에서):
  1. 해당 탭에서 A열 == hanzi인 행을 찾는다 (없으면 `404`).
  2. 모드별 카운트 열(D 또는 E) +1.
  3. `isReview: true`이면 F열 간격을 한 단계 상승시켜 갱신. `isReview: false`인데 이번 정답으로 D≥3 && E≥3이 처음 성립하면(졸업) F열 = `내일|1`.
  4. 7열 이후 첫 빈 셀에 `timestamp|mode` append.
- 응답: 갱신된 해당 단어 객체 (GET과 같은 형태).

**`POST /api/review-fail`** — 복습 오답 (간격 후퇴만, 기록 없음)
- 요청: `{ "tab": "HSK4", "hanzi": "经济" }`
- Worker 동작: F열 간격 한 단계 후퇴(최소 1), F열 = `오늘+후퇴 간격|후퇴 간격`.
- 학습 중 단어의 오답은 API 호출 자체가 없다.
- 전송 실패 시 요청(`tab`·`hanzi`)을 재시도 큐(§10)에 적재 후 재전송한다.

**`GET /api/tabs`** — 학습 대상 탭 목록 *(2026-07-19 단어 등록 v1 편입 — 상세 사양 원본: `docs/plans/word-registration-system.md` §6)*
- 이름이 `_`로 시작하지 않는 탭의 이름 배열을 반환.
- 응답 예: `{ "tabs": ["HSK4", "회화수업"] }`

**`POST /api/words/register`** — 단어 등록 (append-only) *(2026-07-19 단어 등록 v1 편입)*
- 요청:
```json
{ "tab": "HSK4", "createTab": false, "words": [ { "hanzi": "经济", "pinyin": "jīngjì", "meaning": "경제" } ] }
```
- Worker 동작 (클라이언트 검증을 신뢰하지 않고 재검증한다):
  1. 스키마 재검증 — 필드 존재, 한자 유니코드 범위, 성조 부호 형식, 배열 내 중복, `words` 배열 최대 100건. 실패·초과 시 `400`.
  2. 탭 이름 규칙 강제: 앞뒤 공백 트림, `_` 시작 차단(§4.1 학습 제외 규칙과 모순), 트림 후 기존 탭과 동일하면 그 탭에 등록.
  3. `createTab: true`이면 이 시점에 탭 생성 (기존 탭의 1행 헤더 복사). 탭 생성은 등록 제출 시점에만 — 빈 탭 방지.
  4. 탭 내 A열 중복인 단어는 스킵. 나머지는 **A·B·C열만** 마지막 데이터 행 아래에 append (`valueInputOption=RAW` — 값을 수식으로 해석하지 않아 수식 인젝션 차단, 기존 학습 기록 쓰기와 동일). D열 이후 불가침, 기존 행·헤더 수정 없음.
- 응답: added/skipped 상세 (건수 + 단어 목록).

### 7.4 동시성·정합성

- 사용자 1인 전제라 락 없음. Worker의 행 갱신은 "읽고→계산→쓰기"로 단순 구현.
- 학습 세션 도중 시트를 직접 편집하는 경우는 지원하지 않는다 (편집했다면 새 세션 시작으로 해결).

## 8. 인증·보안

- 접근 보호: **비밀번호 1개** (`APP_PASSWORD`).
- 첫 접속 시 비밀번호 입력 화면 → `localStorage`에 저장 → 이후 모든 API 호출에 `Authorization` 헤더로 전송.
- `401` 응답을 받으면 저장된 비밀번호를 지우고 입력 화면으로 되돌린다.
- HTTPS는 Cloudflare 기본 제공. 그 이상의 보안(레이트리밋, 세션 만료 등)은 개인용이므로 v1 비범위.

## 9. 화면 명세

UI 언어는 한국어. 모바일 우선 반응형 (기준 뷰포트 ~390px, 데스크톱에서는 중앙 정렬 카드 레이아웃).

### 9.1 홈
- 상단: 오늘 날짜.
- 현황 카드 3개: **복습 대기 n** / **학습 중 n** / **졸업 n** (§5.1 상태 기준, `/api/words` 응답에서 계산).
- 큰 버튼: **학습 시작** → 학습 화면으로. 출제할 문제가 0개면 버튼 비활성 + "오늘 할 것 없음" 표시.
- "학습 시작" 아래 저강조 텍스트 링크 **"단어 등록 ›"** → 단어 등록 화면 (§9.3). *(2026-07-19 추가)*
- 데이터 로딩 실패 시 에러 메시지 + 재시도 버튼.

### 9.2 학습
- 상단: 진행도 (`12 / 60`) + 종료(홈으로) 버튼.
- 본문: 현재 문제 카드 (§5.2의 모드별 플로우).
  - 모드 1: 한자는 화면에서 가장 큰 요소로. 공개 후 병음·뜻 표시, O/X 버튼은 엄지 닿는 하단에 크게.
  - 모드 2: 뜻 표시 + 텍스트 입력란(자동 포커스) + 제출 버튼. 채점 결과와 정답 정보 표시 후 "다음" 버튼.
- 기록 전송은 문제 단위 비동기(§10). 전송 실패가 학습 진행을 막지 않는다.
- 큐 소진 시 완료 화면: 이번 세션 정답/오답 수, "홈으로" 버튼.

### 9.3 단어 등록 *(2026-07-19 v1 편입 — 상세 사양 원본: `docs/plans/word-registration-system.md` §5)*
- 진입: 홈의 "단어 등록 ›" 링크. 인증은 기존 비밀번호 Bearer 재사용 (§8).
- 플로우:
  1. textarea에 추출 결과(스키마 JSON) 붙여넣기 — 폰에서 claude.ai 앱 복사 → 붙여넣기가 주 시나리오 (모바일 우선).
  2. 클라이언트 검증: JSON 파싱, 필드 존재, 한자 유니코드 범위, 성조 부호 형식, 빈 값, 입력 내 중복 + `pinyin-pro`로 병음-한자 일치 검증. 불일치 행은 차단(수정 강제), 다음자(多音字)는 복수 후보 중 하나만 맞으면 통과. 시트 내 중복은 `GET /api/words` 응답과 대조 (선택 탭 기준).
  3. 검증 결과 테이블: 정상 / 오류(차단) / 중복 행 구분 표시.
  4. 탭 선택 드롭다운 + "+ 새 탭" (이름 입력만, 생성은 제출 시 Worker가 수행 — §7.3).
  5. 중복 행이 있으면 제출 전 명시적 확인 단계 ("중복 n건은 건너뜁니다") — 기존 행은 어떤 경우에도 수정하지 않는다.
  6. 제출 → `POST /api/words/register` → 결과 보고 (추가 n건 / 중복 스킵 n건 / 실패 사유).
- AI 검토 단계 없음 (`pinyin-pro` 사전 검증으로 대체). 등록 취소 기능 없음 — 실수는 구글 시트에서 직접 행 삭제.

## 10. 비기능 요구사항

- **기록 유실 방지**: `POST /api/answer`·`POST /api/review-fail` 실패(네트워크 등) 시 요청을 종류 태그를 붙여 `localStorage` 재시도 큐에 넣고, 앱 로드 시·다음 성공 시점에 적재 순서대로(FIFO) 재전송한다. answer는 타임스탬프(최초 발생 시각)를 그대로 보존해 재전송하고, review-fail은 간격 후퇴 재요청(`tab`·`hanzi`)을 재전송한다.
- **성능**: 단어 수천 개 규모까지는 `/api/words` 한 방 로드로 충분. 페이지네이션 불필요.
- **폰트**: 한자 표시는 간체 우선 (`lang="zh-Hans"` 지정). Noto Sans KR/SC 웹폰트 채택(Google Fonts CDN, 시스템 폰트 폴백). *(2026-07-17 디자인 확정으로 변경, `docs/design-prd.md` §1.2)*
- **입력**: 모드 2 입력란에서 브라우저 자동완성·자동수정 끔 (`autocomplete="off"` 등). 중국어 IME 설치는 사용자 책임.

## 12. 결정 이력 및 기본값 주의사항

인터뷰로 확정된 사항 외에, 아래 항목은 **작성자(Claude)가 기본값으로 정했다.** 구현 전 이견 있으면 이 문서를 고칠 것.

| 항목 | 기본값 | 근거 |
|---|---|---|
| 세션 큐 우선순위 | 복습 대기 → 학습 중 (§6.1) | 복습 누락 방지가 우선 |
| 학습 중 단어 출제 순서 | 총 정답 수 많은 순 | 거의 끝난 단어부터 졸업시키기 |
| F열에 간격 병기 (`날짜\|간격`) | 채택 | 오답 후퇴를 정확히 구현하기 위한 최소 확장. 인터뷰에서는 "6열=다음복습일"까지만 합의됨 |
| 30일 이후 처리 | 30일 간격 무한 반복 | "마스터 졸업" 개념은 v1 불필요 |
| 복습 오답의 세션 내 재출제 | 안 함 | 간격 후퇴로 이미 처리됨, 규칙 단순화 |
| 학습 중 오답의 세션 내 재출제 | 큐 뒤에 1회 | 즉시 재노출이 암기에 유리 |
| 복습 문제 모드 | 무작위 | 인터뷰 합의("양모드 복습")의 구체화 |
| 학습 중 단어의 세션당 문제 수 | 단어당 1문제 — 둘 다 미달이면 모드 무작위 | 소유자 결정(2026-07-19, #44). 종전 "단어당 최대 2문제"는 같은 세션 중복 출제로 폐기 |
| 헤더 행 | 각 탭 1행은 스킵 | 시트 관리 편의 |
| `_` 접두 탭 제외 | 채택 | 메모용 탭 여지 |
| 시간대 | Asia/Seoul | 사용자 위치 |

### 인터뷰 확정 사항 요약
- 개인용 1인 서비스, 비밀번호 1개 보호
- 단어 원천 = 구글 시트, 탭 = 출처별 분류, 학습은 항상 전체 탭 통합 (등록 경로는 2026-07-19 개정 — claude.ai 추출 + 사이트 등록 화면, `docs/plans/word-registration-system.md`)
- 읽기·쓰기 모두 백엔드(Cloudflare Worker + 서비스 계정) 경유
- 모드 1 = 한자→뜻 자가판정(O/X), 모드 2 = 뜻→한자 입력·정확일치 채점
- 졸업 조건 = **모드별** 3회 정답, 오답 리셋 없음
- 졸업 후 = 고정 간격(1·3·7·14·30일) 반복, 양모드 출제, 오답 시 한 단계 후퇴
- 시트 컬럼: A한자 B병음 C뜻 D모드1정답수 E모드2정답수 F다음복습일 G~타임스탬프(정답만)
- 세션 = 최대 60문제, 모드 자동 혼합
- 스택 = React(Vite) + Cloudflare Workers, 모바일 우선 반응형, 화면은 홈+학습 2개 (2026-07-19 개정: 단어 등록 화면 추가로 3개)
