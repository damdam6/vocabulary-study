1요약

Cloudflare 스캐폴딩 데모 페이지·자산·CSS를 전부 제거하고, docs/design-prd.md §1이 정의한 공통 기반 — 다크 테마 색 토큰(CSS 변수), Noto Sans KR/SC 웹폰트, 430px 중앙 컬럼 레이아웃, 공통 애니메이션(popfb/fadeup/shake) — 을 글로벌 스타일로 구축한다. 그 위에 상태 기반 4화면(로그인/홈/학습/완료) 전환 셸을 플레이스홀더 컴포넌트로 세워, 후속 프론트 이슈 #11(로그인)·#13(홈)·#14(세션 큐)·#15(학습 셸)가 병렬로 착수할 수 있게 한다.

2현황 분석

  • 데모 페이지: src/App.tsx:1-148이 Cloudflare 스캐폴딩 그대로다 — 로고 3종 + hero.png import(src/App.tsx:2-5), 카운터 버튼, /api/health 체크 버튼(src/App.tsx:39), Vite 문서/소셜 링크 섹션. 전용 스타일 src/App.css(184줄)도 데모 전용.
  • 데모 자산: src/assets/의 react.svg·vite.svg·cloudflare.svg·hero.png 4개. public/icons.svg는 데모 페이지만 쓰는 아이콘 스프라이트(src/App.tsx:58 등 /icons.svg#… 참조). public/favicon.svg는 Vite 보라색 로고로 역시 데모 자산이며 index.html:5가 참조 중.
  • 글로벌 스타일: src/index.css:1-51이 스캐폴딩의 라이트/다크 자동 전환 토큰과 1126px 컬럼(src/index.css:53-63)을 정의 — design-prd(다크 고정, 430px 컬럼)와 전면 배치된다. 웹폰트·공통 애니메이션·화면 전환 구조 없음.
  • index.html: lang="en"(index.html:2), title vocabulary-study(index.html:7), theme-color·폰트 링크 없음.
  • 디자인 기준: docs/design-prd.md §1.1(색 토큰 17종) · §1.2(폰트 스택, 웹폰트 로딩 방식) · §1.3(애니메이션 3종 — popfb는 목업 0.68s에서 0.4s로 단축 확정) · §1.4(레이아웃). 목업 원본 docs/design/prototype.dc.html:16-27에 글로벌 스타일·keyframes·컬럼 구조의 참조 구현이 있다. 단 목업 폰트 링크(prototype.dc.html:14)에 포함된 Noto Serif SC는 §8에서 기각된 옵션이므로 제외해야 한다.
  • 백엔드는 완비: worker/index.ts에 /api/health·/api/words·/api/answer·/api/review-fail이 이미 구현돼 있다. 이 이슈는 프론트 셸만 다루며 worker는 건드리지 않는다.
  • 빌드 체인: tsconfig.app.json에 noUnusedLocals/noUnusedParameters가 켜져 있고(tsconfig.app.json:22-23), lint는 oxlint(react 플러그인), 테스트는 vitest(src/lib/wordState.test.ts 기존 테스트 존재). CI는 main 푸시 시 배포만 하므로(.github/workflows/deploy.yml) 검증은 로컬 build/lint/test가 기준.

3설계

  • 글로벌 스타일 = src/index.css 전면 재작성 (파일 추가 없이 단일 파일 유지):
    • :root에 §1.1 토큰 17종을 CSS 변수로 정의 — --bg-page: #0b0d12, --bg-app: #10131a, --bg-card, --bg-panel, --bg-disabled, --border-faint, --border-card, --border-input, --text, --text-muted, --text-faint, --text-placeholder, --text-hint-dim, --accent, --accent-light, --green, --red. color-scheme: dark로 다크 고정(라이트 모드 없음 — §1.1).
    • body 기본: margin 0, var(--bg-page) 배경, var(--text), UI 폰트 스택 'Noto Sans KR', -apple-system, 'Apple SD Gothic Neo', sans-serif, 안티앨리어싱 (prototype.dc.html:16 이식).
    • :lang(zh-Hans) 셀렉터로 한자 폰트 'Noto Sans SC', 'Noto Sans KR', sans-serif 자동 적용 — 후속 이슈들이 lang="zh-Hans" 속성만 붙이면 폰트가 따라오게 해서 §1.2 규칙("한자 표시 요소는 항상 lang=zh-Hans")을 CSS 한 곳으로 수렴.
    • 공용 리셋: input/button font-family 상속, input outline:none + placeholder 색 var(--text-placeholder), button cursor:pointer (prototype.dc.html:18-20).
    • keyframes 3종 popfb/fadeup/shake — 목업 prototype.dc.html:21-23 곡선 그대로. duration은 사용처에서 지정하는 값이므로 keyframes만 정의(§5.1의 popfb 0.4s 단축은 #15가 사용 시점에 적용).
    • 레이아웃 클래스 .app-frame(min-height 100dvh, flex 중앙, --bg-page) + .app-column(width 100%, max-width 430px, min-height 100dvh, flex column, --bg-app, 좌우 1px --border-faint, position relative — 피드백 오버레이 #15의 앵커) — prototype.dc.html:26-27 구조 이식.
  • 화면 전환 = useState 기반: src/App.tsx가 type Screen = 'login' | 'home' | 'study' | 'done' 상태를 보유하고 전환 콜백을 플레이스홀더에 내려준다. 전환 그래프: login→home(로그인 성공) · home→study(학습 시작) · study→home(종료) · study→done(세션 완료) · done→home(홈으로). 기각: react-router(4화면 SPA에 과함, 이슈가 "상태 기반" 명시), Context/전역 스토어(콜백 5개는 props로 충분 — 필요해지면 후속 이슈에서 도입).
  • 플레이스홀더 4개 = src/screens/: LoginScreen/HomeScreen/StudyScreen/DoneScreen. 각각 화면명 텍스트 + 전환 버튼만 가진 최소 구현으로, 파일 경로와 props 시그니처(onLogin/onStart/onExit·onComplete/onGoHome)를 이 이슈에서 확정하고 내부는 #11/#13/#15가 채운다. StudyScreen 플레이스홀더에 lang="zh-Hans" 한자 샘플 한 줄을 넣어 완료 조건의 폰트 적용 확인이 셸 단계에서 가능하게 한다.
  • 웹폰트 = index.html: preconnect 2건(googleapis/gstatic) + css2?family=Noto+Sans+KR:wght@400;500;700&family=Noto+Sans+SC:wght@400;500;700&display=swap. 목업 링크에서 Serif SC만 제거한 형태. CDN 실패 시 폴백 스택으로 동작(§1.2). 기각: 셀프 호스팅 서브셋(§8 — 시트에 단어가 계속 추가돼 정적 서브셋 위험).
  • favicon 교체: public/favicon.svg(Vite 로고)를 design-prd §1.1 로고 사양 축소판 — 둥근 사각형 linear-gradient(135deg,#2f63e6,#5b8bf5) + 흰색 "词" — 의 미니 SVG로 교체. 단순 삭제하면 index.html:5 참조가 404가 되고, 유지하면 "빌드 결과에 데모 자산 없음" 완료 조건에 걸리므로 교체가 맞다. (이슈 본문에 없는 결정 — PR에서 명시할 것.)

4작업 단계

  1. 데모 파일 삭제 — 데모 페이지 전용 CSS·자산·아이콘 스프라이트를 git rm src/App.css, src/assets/react.svg, src/assets/vite.svg, src/assets/cloudflare.svg, src/assets/hero.png, public/icons.svg
  2. favicon 교체 — Vite 로고를 "词" 그라데이션 로고 SVG로 교체 (§1.1 로고 사양 축소판) public/favicon.svg
  3. index.html 재작성 — lang="ko", <title>단어 암기</title>, <meta name="theme-color" content="#0b0d12">, preconnect 2건 + Noto Sans KR/SC 400·500·700 display=swap 폰트 링크 index.html
  4. 글로벌 스타일 재작성 — §1.1 토큰 CSS 변수, color-scheme: dark, body 기본·폰트 스택, :lang(zh-Hans) 한자 폰트, input/button 리셋, popfb/fadeup/shake keyframes, .app-frame/.app-column 레이아웃 클래스 src/index.css
  5. 화면 플레이스홀더 4개 생성 — 화면명 + 전환 버튼 최소 구현, props 시그니처 확정. StudyScreen에 lang="zh-Hans" 한자 샘플 포함 src/screens/LoginScreen.tsx, src/screens/HomeScreen.tsx, src/screens/StudyScreen.tsx, src/screens/DoneScreen.tsx
  6. App.tsx 재작성 — Screen 상태 + 전환 콜백, .app-frame/.app-column 래핑, 데모 import 전부 제거 (src/main.tsx는 변경 없음) src/App.tsx
  7. 검증 — npm run build·lint·test + dist 데모 자산 부재 확인 + dev 서버 수동 확인(테스트 계획 참조) dist/client/ (검사 대상)

5리스크

리스크영향완화책
noUnusedLocals/noUnusedParameters(tsconfig.app.json:22-23)에 플레이스홀더의 미사용 props가 걸림tsc -b 빌드 실패모든 전환 콜백 props를 실제 버튼 onClick에 연결 — 전환 동작 확인(완료 조건 4)과도 일치
후속 이슈 #11/#13/#15가 같은 src/screens/* 파일을 병렬로 채움머지 충돌·시그니처 불일치플레이스홀더를 최소로 유지하고 props 시그니처를 이 PR에서 확정, 이슈 코멘트로 파일 경로·시그니처 공지
SVG favicon의 <text> "词" 렌더링이 브라우저의 시스템 CJK 폰트에 의존일부 환경에서 글자 누락글자가 빠져도 그라데이션 사각형은 표시됨 — 식별성 유지. 문제 시 path 아웃라인으로 후속 교체 가능
Google Fonts CDN 차단/실패웹폰트 미적용§1.2 설계 그대로 — display=swap + 시스템 폴백 스택이라 기능 지장 없음. 폴백 동작을 테스트 계획에 포함
목업의 Noto Serif SC·m1AutoMs 등 기각된 사양을 그대로 이식할 위험확정 사양과 불일치design-prd §6 "목업 대비 변경점" 표를 구현 중 체크리스트로 사용 (이 이슈 범위에선 Serif 제거만 해당)
