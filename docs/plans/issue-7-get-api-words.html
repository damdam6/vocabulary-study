#7기능 개선

GET /api/words 구현 — 전 탭 통합 단어 목록 조회

damdam6/vocabulary-study 브랜치 feat/7-get-api-words ← main 라벨 enhancement 2026-07-15

1요약

GET /api/words 엔드포인트를 신설해, _로 시작하지 않는 모든 탭의 단어를 하나의 words 배열로 통합 반환한다. D·E열 빈칸은 0, F열 빈칸은 { nextReview: null, interval: null }로 정규화한다. 큐 구성·상태 판정은 클라이언트 책임이므로 이 Worker는 시트 값을 그대로 중계·정규화만 한다. 의존 이슈 #5(구글 시트 API 연동)는 이미 머지돼 worker/lib/sheets.ts·worker/lib/google-auth.ts를 그대로 재사용할 수 있다.

2현황 분석

  • worker/index.ts:1-10 — 라우팅은 /api/health 하나뿐이고 404 폴백만 있음. fetch(request) 시그니처에 env 파라미터가 없어 시트 접근 불가 — 이번 작업에서 fetch(request, env)로 확장 필요.
  • worker/lib/sheets.ts:34-38 — getValues(env, tab, range)는 탭+범위 하나를 읽어 string[][]를 반환. 뒤쪽 빈 셀은 API가 생략하므로 행마다 길이가 다를 수 있음(주석에 명시). 하지만 탭 이름 목록을 얻는 함수가 없다 — 스프레드시트 메타데이터 조회(GET /v4/spreadsheets/{id}?fields=sheets.properties.title) 래퍼를 새로 추가해야 전체 탭을 순회할 수 있다.
  • worker/lib/sheets.ts:78-97 — sheetsFetch(env, method, path, body)는 path를 그대로 {API_BASE}/{SHEET_ID}{path}에 붙인다. 값 범위가 아닌 메타데이터 조회는 path = "?fields=sheets.properties.title"처럼 쿼리스트링만 넘기면 기존 함수를 그대로 재사용 가능(별도 fetch 로직 불필요).
  • worker/lib/google-auth.ts — 토큰 발급·캐시는 이미 완성돼 있어 이번 이슈에서 손댈 부분 없음.
  • PRD §7.3 응답 예시의 fetchedAt: "2026-07-13 09:00"는 ISO가 아니라 §5.4에 정의된 YYYY-MM-DD HH:mm(Asia/Seoul) 형식과 동일 — 이 포맷터가 아직 코드에 없음. 같은 포맷이 이후 이슈(#8 POST /api/answer의 타임스탬프)에도 재사용되므로 공용 유틸로 분리하는 편이 합리적.
  • PRD §4.2 — F열 계약은 YYYY-MM-DD|일수(예: 2026-08-12|14). 빈칸이면 미졸업 단어. 이슈 완료 조건은 이 정규화를 실제 시트의 HSK6급 탭에 테스트 행을 추가해 확인하도록 요구함.
  • 이슈 #6(API 비밀번호 인증 미들웨어)은 별도 이슈로 아직 미착수 상태(sibling worktree .worktrees/6-api-password-auth-middleware 존재) — #7은 #5에만 의존하므로 이번 엔드포인트는 인증 없이 구현하고, #6이 머지되면 자동으로 보호 대상에 포함된다.
  • 로컬 시크릿 — 이전 이슈(#5) 플랜 문서에 언급된 .dev.vars·.secret/ 키 파일이 현재 이 worktree와 메인 체크아웃 어디에도 없음(git-ignore 대상이라 worktree마다 재생성 필요). 실제 시트를 상대로 한 수동 테스트 전에 재준비가 필요 — 리스크 절 참고.

3설계

  • 탭 목록 조회 — sheets.ts에 getSheetTitles(env): Promise<string[]> 추가. sheetsFetch(env, "GET", "?fields=sheets.properties.title") 호출 후 sheets[].properties.title만 뽑아 배열로 반환. 기존 4개 함수와 이름 패턴 (get/update/append) 유지.
  • 탭 필터링 — 라우트 레벨에서 title.startsWith("_")인 탭 제외 (PRD §4.1).
  • 행 조회 범위 — 탭마다 getValues(env, tab, "A2:F"). G열 이후(타임스탬프)는 이 응답에 불필요하므로 읽지 않음 — 응답 크기·API 호출 비용 최소화.
  • 동시성 — 탭별 조회는 서로 독립적이므로 Promise.all로 병렬 실행. 개인용 소규모 스프레드시트(탭 수 적음) 전제라 별도 values:batchGet 배치 API 도입은 지금 단계에서 과함 — 필요해지면 나중에 전환.
  • 행 → 단어 정규화 — 새 worker/lib/words.ts에 WordEntry 타입과 parseWordRow(tab, row): WordEntry를 둔다. routes/가 아니라 lib/에 두는 이유: PRD §7.3 POST /api/answer 응답이 "GET과 같은 형태"라고 명시돼 있어 #8 구현이 이 정규화 로직을 그대로 import해 재사용해야 함.
    • A/B/C(한자/병음/뜻) → hanzi/pinyin/meaning 그대로.
    • D/E 빈칸 또는 미존재(trailing 셀 생략) → 0.
    • F 빈칸/미존재 → { nextReview: null, interval: null }. 값이 있으면 "|" 기준 분리해 앞부분은 그대로, 뒷부분은 Number(). 형식이 깨진 값(수동 편집 등)은 throw하지 않고 null 쌍으로 방어적 처리.
  • 시간 유틸 — 새 worker/lib/time.ts에 formatSeoulDateTime(date = new Date()): string. Intl.DateTimeFormat + timeZone: "Asia/Seoul" + formatToParts로 YYYY-MM-DD HH:mm 조립(로케일 문자열 파싱에 의존하지 않아 포맷이 고정됨). #8 타임스탬프 기록에도 재사용 예정.
  • 라우트 — 새 worker/routes/words.ts에 handleGetWords(env): Promise<Response>. 탭 목록 → 필터 → 병렬 read → flatMap 정규화 → { fetchedAt, words } JSON 응답.
  • worker/index.ts 변경 — fetch(request, env)로 시그니처 확장, GET /api/words 라우트 추가 후 handleGetWords 위임. 시트 호출은 실패할 수 있으므로(SheetsApiError, 토큰 발급 실패 등) 라우트 핸들러를 try/catch로 감싸 미처리 예외 대신 500 JSON 에러로 응답(현재는 이런 방어가 전혀 없어 실패 시 Workers 기본 에러 페이지가 노출됨).

4작업 단계

  1. 스프레드시트 메타데이터(탭 제목 목록) 조회 함수 getSheetTitles 추가 worker/lib/sheets.ts
  2. Asia/Seoul YYYY-MM-DD HH:mm 포맷터 추가 worker/lib/time.ts (신규)
  3. WordEntry 타입 + 시트 행 → 단어 객체 정규화 함수(D/E 기본값 0, F열 날짜|간격 파싱, 방어적 처리) 작성 worker/lib/words.ts (신규)
  4. GET /api/words 핸들러: 탭 목록 조회 → _ 접두 탭 제외 → 탭별 A2:F 병렬 read → 정규화해 { fetchedAt, words } 응답 조립 worker/routes/words.ts (신규)
  5. fetch 시그니처에 env 추가, GET /api/words 라우팅 연결, 시트 호출 실패 시 500 JSON 응답으로 감싸기 worker/index.ts
  6. 로컬 시크릿(.dev.vars) 준비 후 wrangler dev로 실제 시트를 상대로 수동 검증 (§6) — 테스트용 행/탭은 검증 후 원상복구 .dev.vars (미커밋, 로컬 전용)

5리스크

리스크영향완화책
이 worktree에 .dev.vars/서비스 계정 키 파일이 없음(git-ignore 대상, worktree별로 별개)실제 시트를 상대로 한 수동 테스트 착수 지연메인 체크아웃이나 안전한 보관소에서 키를 다시 가져와 .dev.vars 재생성 — 커밋 금지, 작업 후 git status로 미포함 확인
탭 수가 늘어나면 Promise.all 동시 요청이 늘어나 Sheets API 분당 요청 한도에 근접대량 탭 환경에서 429 가능성개인용 소규모 스프레드시트라 v1 범위에선 무시 가능. 문제 발생 시 values:batchGet 단일 요청으로 전환
F열이 수동 편집 등으로 날짜|간격 형식을 벗어남파싱 실패 시 전체 응답이 500으로 죽을 수 있음파싱 실패를 throw 대신 null 쌍으로 방어적 처리(작업 단계 3)
#6(인증 미들웨어) 미착수 상태에서 #7이 먼저 머지되면 /api/words가 일시적으로 무인증 공개배포 시점에 시트 데이터가 비밀번호 없이 노출될 수 있음이슈 의존관계상 의도된 순서(#7은 #5에만 의존) — 실제 배포는 #6 머지 이후로 조율하거나, 배포 전 #6을 먼저 머지

6테스트 계획

  • 기본 응답: wrangler dev 실행 후 curl localhost:8787/api/words | jq → fetchedAt이 YYYY-MM-DD HH:mm 형식, words가 기존 실제 탭들의 데이터를 담고 있는지 확인.
  • D/E/F 정규화(완료 조건 1): HSK6급 탭에 테스트 행 2개 추가 — (a) D/E/F 모두 빈칸, (b) D=3, E=3, F=2026-08-01|7. curl 재호출 → (a)는 m1:0, m2:0, nextReview:null, interval:null, (b)는 m1:3, m2:3, nextReview:"2026-08-01", interval:7로 나오는지 확인.
  • _ 접두 탭 제외(완료 조건 2): _임시 탭을 새로 만들고 단어 행 하나 추가 → curl 재호출 → 응답 words에 해당 탭 데이터가 전혀 없는지 확인.
  • 전 탭 통합(완료 조건 3): 서로 다른 두 개 이상의 탭(예: HSK6급과 기존 다른 탭)에 단어가 있을 때, 응답 words 배열 하나에 tab 필드로 구분되어 모두 섞여 나오는지 확인.
  • 정리: 위 테스트용 행·탭은 검증 직후 시트에서 삭제해 원상복구.
  • 회귀: /api/health가 여전히 200 JSON 반환, npm run build (tsc -b 포함)·npm run lint 클린.

7완료 조건

  • HSK6급 탭에 D/E/F 각각 빈칸인 테스트 행과 값이 채워진 테스트 행을 추가해 정규화 결과(0/0/null/null vs 실제 값) 확인
  • _로 시작하는 임시 탭을 만들어 결과에서 제외되는지 확인
  • 여러 탭에 걸친 단어가 하나의 words 배열로 통합되는지 확인
  • worker/lib/words.ts의 정규화 함수가 #8(POST /api/answer)에서 import만으로 재사용 가능한 형태로 분리돼 있음
  • 테스트용 행/탭 원상복구, 시크릿 미커밋(git status 확인), lint·build 통과
