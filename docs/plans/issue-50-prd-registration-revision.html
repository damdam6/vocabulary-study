1요약

단어 등록 시스템 플랜(docs/plans/word-registration-system.md) 확정으로 등록 경로가 "별도 Claude skill"에서 "claude.ai 추출 + 사이트 등록 화면"으로 바뀌었으나, docs/PRD.md는 여전히 구 구조를 서술하고 있다. PRD의 8개 절(§1·§2·§3·§4·§7.1·§7.3·§9·§11·§12)을 플랜과 모순 없게 개정한다. 원칙: 상세 사양의 원본은 플랜 문서, PRD는 요약 + 참조. 시트 컬럼 계약(§4) 자체는 불변 — "쓰는 주체" 표기만 갱신한다. 코드 변경 없음, 구현은 #47(킷)·#48(Worker API)·#49(등록 화면) 몫.

3설계

  • 단일 원본 원칙: 등록 파이프라인의 상세 사양(스키마, 검증 규칙, 탭 이름 규칙, 중복 정책)은 플랜 §3·§5·§6이 원본. PRD에는 동작 요약 + docs/plans/word-registration-system.md 참조 링크만 싣는다 — 이중 서술로 인한 drift 방지.
  • §4 계약 불변: 컬럼 정의·형식·주의사항은 한 글자도 바꾸지 않는다. 바뀌는 것은 "쓰는 주체" 컬럼의 표기(등록 skill → 사이트의 등록 화면 경유)와 도입부의 계약 당사자 서술뿐. v1에서 시트 쓰기 주체는 사이트(Worker)로 일원화된다.
  • 비범위 재정의: 등록 UI는 v1 포함으로 이동하되, 편집·삭제는 여전히 비범위(플랜 §8 Q6 — 등록 취소 기능 없음, 실수는 시트에서 직접 삭제). §3 비범위 문구를 "단어 편집·삭제 UI"로 좁힌다.
  • 이력 보존: §12는 결정 이력 절이므로 인터뷰 확정 사항을 삭제하지 않고 개정 주석을 병기한다. 단, "skill" 단어 자체는 개정 문구로 대체해 grep 완료 조건을 만족시킨다.
  • 개정 표기: 문서 머리(docs/PRD.md:6)에 개정일 추가, 본문 변경점에는 기존 관행대로 "(2026-07-19 단어 등록 v1 편입, docs/plans/word-registration-system.md)" 인라인 주석.
