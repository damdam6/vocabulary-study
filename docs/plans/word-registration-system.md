# 단어 등록 시스템 플랜

> 캡처/깨진 텍스트에서 단어를 뽑아 구글 시트에 등록하는 전체 파이프라인.
> 작성일: 2026-07-19 · 상태: **확정** (grill 세션으로 미결 Q1~Q6 해소, 결정 로그 §8)

---

## 1. 배경

- PRD v1은 단어 등록을 "별도 Claude skill" 몫으로 비범위 처리했다 (§3). 이번에 등록 경로를 **사이트 안으로** 들여온다 → PRD 개정 필요.
- 요구사항: 폰·노트북·다른 데스크탑 어디서나 동일하게 동작. 자격증명(서비스 계정 키)은 Worker에만 존재하고 기기·프롬프트에 배포하지 않는다.
- 검토 후 기각한 대안: 로컬 킷 폴더(멀티 디바이스 불가), Worker MCP 커넥터(어드민 페이지가 더 단순·범용), AI 검토 단계(§8 Q3).

## 2. 전체 구조 (3단계)

```
[1. 추출 — claude.ai 프로젝트 (사용자 개인 세팅, 리포 밖)]
   이미지·깨진 텍스트 → 간체자/병음/한글 스키마 JSON
   (schema_check.py로 출력 자체 검증) → 사용자가 결과 복사

[2. 등록 — vocaStudy 단어 등록 화면 (이 리포)]
   붙여넣기 → 파싱 + 기계 검토(pinyin-pro 포함)
   → 탭 선택/생성 → (중복 있으면 확인) → 제출

[3. 저장 — Cloudflare Worker (이 리포)]
   재검증 · 중복 스킵 · append-only 쓰기 → 구글 시트
```

- **신뢰 경계**: 1단계 출력은 신뢰하지 않는다 — 2단계(클라이언트)와 3단계(Worker)가 각각 재검증. 시트 계약(append-only, D열 이후 불가침, 헤더 불변)의 최종 강제는 Worker 구현이 담당.
- **채팅 독립성**: 인터체인지가 스키마 JSON이므로 추출 채팅은 claude.ai가 기본값일 뿐, 스키마만 지키면 다른 챗으로 대체 가능. 사이트는 출처를 모른다.
- **API 비용 0**: 추출은 claude.ai 구독 내 채팅. 검증은 사전 라이브러리(결정적). 유료 API 호출 없음.

## 3. 데이터 스키마 (확정)

```json
{
  "version": 1,
  "words": [
    { "hanzi": "经济", "pinyin": "jīngjì", "meaning": "경제" }
  ]
}
```

- **JSON 확정 이유**: 채팅 복사 시 탭·공백 변형에 안전, 파싱 무결. (TSV 기각 — 복사 과정 탭 문자 유실 위험)
- 필드 규칙: `hanzi` 간체·배열 내 중복 금지, `pinyin` 성조 부호 필수(`jīngjì`, 숫자 표기 불가), `meaning` 한국어 간결.
- **탭 이름은 스키마에 없다** — 등록 화면에서만 선택 (캡처에는 분류 정보가 없어 Claude 추측은 무의미).
- 스키마 정의는 이 문서가 원본. `schema_check.py`(1단계) · 사이트 TS 검증(2단계) · Worker 검증(3단계)이 이를 따르고, 각 구현에 원본 참조 주석을 남긴다 (drift 방지).

## 4. Claude 프로젝트 킷 (작업 b)

claude.ai 프로젝트에 올릴 산출물 2개 (리포 `docs/registration-kit/`에 보관):

1. **지침 프롬프트** (영어) — 기존 작성본 개정: 이미지/깨진 텍스트 파싱 지침 유지, 시트 접근·연결 정보 관련 내용 전부 제거, 출력 = §3 스키마 JSON 코드블록 하나로 고정. 출력 전 `schema_check.py` 실행해 자체 검증하도록 지시.
2. **`schema_check.py`** — 스키마 검증기 (필드 존재, 한자 유니코드 범위, 성조 부호 형식, 배열 내 중복). claude.ai 코드 실행으로 Claude가 스스로 돌리는 용도.

## 5. 단어 등록 화면 (작업 c, d — 확정 사양)

- **진입**: 홈 "학습 시작" 아래 저강조 텍스트 링크 "단어 등록 ›". `App.tsx`의 `Screen` 유니온에 `'register'` 추가 (라우터 없음 유지).
- **인증**: 기존 `APP_PASSWORD` Bearer 재사용.
- **플로우**:
  1. textarea 붙여넣기 (모바일 우선 — 폰 Claude 앱 복사 → 붙여넣기가 주 시나리오)
  2. 파싱 + 기계 검토:
     - 형식: JSON 파싱, 필드 존재, 한자 유니코드 범위, 성조 부호 형식, 빈 값, 입력 내 중복
     - **병음-한자 일치: `pinyin-pro` 사전 검증. 불일치 행은 차단(수정 강제)** — 다음자(多音字)는 복수 후보 중 하나만 맞으면 통과. 수정은 붙여넣은 JSON을 고치거나 추출 단계로 돌아가서.
     - 시트 내 중복: 기존 `GET /api/words` 응답과 대조 (선택 탭 기준)
  3. 검증 결과 테이블: 정상 / 오류(차단) / 중복 행 구분 표시
  4. 탭 선택 드롭다운 + "+ 새 탭" (이름 입력만, 생성은 제출 시)
  5. **중복 행이 있으면 제출 전 명시적 확인 단계** ("중복 n건은 건너뜁니다" 확인) — 기존 행은 어떤 경우에도 수정하지 않음
  6. 제출 → Worker → 결과 보고 (추가 n건 / 중복 스킵 n건 / 실패 사유)
- **AI 검토 없음** (기각, §8 Q3). **등록 취소 기능 없음** — 실수는 구글 시트에서 직접 행 삭제 (§8 Q6).

## 6. Worker API (작업 e — 확정 사양)

| 엔드포인트 | 동작 |
|---|---|
| `GET /api/tabs` | 학습 대상 탭 목록 (`_` 접두 제외) |
| `POST /api/words/register` | `{ tab, createTab?, words[] }` → 스키마 재검증(`words` 최대 100건, 초과 `400`), 탭 내 중복 스킵, A·B·C열만 마지막 데이터 행 아래 append (`valueInputOption=RAW` — 수식 인젝션 차단). `createTab`이면 기존 탭 1행 헤더 복사해 생성 (탭 생성은 이 시점에만 — 빈 탭 방지). 응답: added/skipped 상세 |

- 탭 이름 규칙 (Worker에서도 강제): 앞뒤 공백 트림, **`_` 시작 차단**(학습 제외 탭과 모순), 트림 후 기존 탭과 동일하면 그 탭에 등록.
- 인증: 기존 Bearer 미들웨어 그대로. `worker/lib/sheets.ts`·`google-auth.ts` 재사용.
- 시트 계약 강제 지점: append-only · D열 이후 불가침 · 헤더 불변은 여기 구현으로 보장.
