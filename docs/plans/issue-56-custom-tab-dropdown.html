등록 화면 탭 드롭다운을 커스텀 컴포넌트로 교체 — 네이티브 select 제거

1요약

등록 화면(src/screens/RegisterScreen.tsx:227)의 탭 선택이 네이티브 <select>라 OS 기본 팝업이 뜨고 디자인 토큰과 이질적이다. 이를 design-prd §1 토큰(bg-panel/border-input/bg-card) 기반 커스텀 드롭다운으로 교체하되, 선택값·"+ 새 탭" 흐름을 담당하는 기존 상태 로직은 그대로 두고 UI 레이어만 바꾼다. 다른 화면에서도 쓸 수 있도록 src/components/Dropdown.tsx에 독립 컴포넌트로 분리하고, 바깥 클릭/ESC 닫힘과 방향키·Enter 키보드 조작을 지원하는 리스트박스 버튼 패턴으로 구현한다.

  • 관련 상태: selectedTab(RegisterScreen.tsx:47), tabs(RegisterScreen.tsx:45, GET /api/tabs 결과), NEW_TAB_VALUE = '__new__'(RegisterScreen.tsx:36).

3설계

  • 패턴 선택: WAI-ARIA "리스트박스 버튼" 패턴 — 트리거 <button aria-haspopup="listbox" aria-expanded> + 열림 시 role="listbox" 패널, 옵션은 role="option" aria-selected.
    기각한 대안 1: 타이핑 자동완성이 가능한 combobox 패턴 — 이슈 요구사항은 방향키·Enter·바깥클릭/ESC뿐이라 검색 입력은 범위 밖의 불필요한 복잡도.
    기각한 대안 2: <details>/<summary> 네이티브 토글 — 옵션별 aria-selected·방향키 이동·선택 시 자동 닫힘을 얹으려면 결국 동일한 커스텀 키보드 핸들러가 필요해 이점이 없고, 브라우저별 열림 애니메이션 제약만 늘어난다.
  • 상태 소유: 컴포넌트 내부에는 open(패널 표시 여부)과 activeIndex(키보드 하이라이트 인덱스)만 로컬로 두고, 선택값 자체는 완전 제어형 — value/onChange props로 부모가 소유한다. RegisterScreen의 selectedTab 등 기존 상태는 손대지 않는다.
  • 바깥 클릭 / ESC: open === true일 때만 document에 mousedown + keydown 리스너를 붙이는 useEffect. 컨테이너 ref 바깥 클릭 또는 Escape에서 패널을 닫고 트리거 버튼으로 포커스를 되돌린다.
  • 키보드: 트리거 포커스 상태에서 ArrowDown/ArrowUp/Enter/Space → 패널 열기(현재 선택값 위치로 activeIndex 초기화). 패널이 열려 있을 때 ArrowDown/ArrowUp → activeIndex 이동(경계에서 정지), Enter/Space → 해당 옵션 선택 후 닫기, Escape → 선택 없이 닫기.
  • 라벨 연결: <button>은 HTML5 labelable element이므로 기존 <label htmlFor="register-tab-select">를 트리거 버튼의 id에 그대로 연결하면 라벨 클릭 시 트리거로 포커스가 이동한다 — 별도 마크업 변경이 필요 없다.
  • Props(최소): id, value: string, options: { value: string; label: string }[], onChange: (value: string) => void. 현재 다른 화면에서 쓰이지 않으므로 disabled·placeholder 등 미사용 옵션은 추가하지 않는다(투기적 설계 지양).
  • 스타일: 트리거는 기존 .register-select 톤(48px 높이 / radius 12px / border-input / bg-panel)을 .dropdown-trigger로 이전. 패널은 .dropdown-panel — bg-card 배경, border-card 테두리, radius 14px, 그림자(문제 카드 그림자 톤 참고), fadeup 0.3s 애니메이션 재사용. 옵션 .dropdown-option은 터치 영역 확보를 위해 최소 높이 44px, hover/active 시 bg-panel 톤 강조, 선택된 옵션은 accent-light 텍스트로 표시.

5리스크

리스크영향완화책
네이티브 <select> 제거로 일부 브라우저/스크린리더의 select 관용구(OS 자동완성, 접근성 단축키)를 잃는다보조기술 사용자 경험이 네이티브 대비 저하될 수 있음ARIA 리스트박스 패턴(role/aria-selected/aria-expanded/aria-haspopup)을 정확히 준수해 스크린리더 대응. 이슈 목적(디자인 토큰 통일)상 시각적 일관성이 우선
바깥 클릭 감지(document mousedown)가 등록 화면의 다른 인터랙티브 요소(중복 확인 배너 버튼 등)와 타이밍이 겹칠 수 있음드롭다운이 의도치 않게 안 닫히거나 다른 클릭을 가로챌 가능성컨테이너 ref 기반 판정 + 패널이 열려 있을 때만 리스너를 부착·해제해 영향 범위를 최소화
