퀴즈 모드 1 UI — 한자→뜻 자가판정

1요약

#15에서 셸과 판정 계약(onJudged)까지 완성된 모드 1 플레이스홀더 카드를 design-prd §4.2의 확정 사양으로 교체한다. 핵심은 3D 플립 카드(앞면 한자만 → 탭 시 rotateX 0.55s 플립 → 뒷면 한자+병음+뜻), 글자 수 적응형 한자 크기(84/64/52px), 공개 후 fadeup으로 등장하는 82px 원형 O/X 버튼(X 왼쪽·O 오른쪽)이다. 진행·기록·피드백 오버레이는 전부 셸 소유이므로 이 이슈는 Mode1Card.tsx 한 컴포넌트와 CSS만 건드린다.

2현황 분석

셸(#15, 머지됨)이 이미 오케스트레이션을 전부 소유한다:

  • src/screens/StudyScreen.tsx:131-132 — 모드 1 문제면 <Mode1Card key={session.pos} question onJudged={handleJudged}/>로 렌더. 문제마다 key가 바뀌어 리마운트되므로 카드 내부 상태(플립 여부)는 초기화를 신경 쓸 필요 없다.
  • src/screens/StudyScreen.tsx:86-98 — handleJudged(correct)가 기록 effect 발사 + O/X 오버레이 재생 + 즉시 다음 문제 전환까지 처리. 카드는 O/X 탭 순간 onJudged를 한 번 호출하고 끝나면 된다(대기 없음 — 목업의 700ms 딜레이는 이슈 코멘트에서 기각 확정).
  • src/screens/Mode1Card.tsx — 현재 플레이스홀더: 플립 없이 같은 카드 안에서 내용만 교체(revealed state), O/X는 직사각형 버튼(O가 오른쪽인 것은 이미 맞음), 한자 크기는 44px 고정.
  • src/index.css:325-430 — 플레이스홀더 스타일. .mode-card·.mode-card-hanzi(44px)·.mode-card-pinyin(20px)·.mode-card-meaning(23px)·.mode-card-hint는 Mode2Card(#17)도 공유하고, .judge·.mode-footnote는 모드 1 전용.
  • src/index.css:108-117 — fadeup keyframe은 이미 존재(0.3s는 사용처 지정). flip은 keyframe이 아니라 transform transition으로 구현하는 게 목업 방식(docs/design/prototype.dc.html:82).
  • src/index.css:40-42 — :lang(zh-Hans) 셀렉터가 한자 폰트(Noto Sans SC 웹폰트+폴백)를 일괄 적용. 한자 요소에 lang="zh-Hans"만 달면 된다(완료 조건의 폰트 항목은 이 경로로 충족, 이슈 코멘트가 "시스템 폰트" 문구를 웹폰트 사양으로 대체 확정).
  • 목업 원본 docs/design/prototype.dc.html:81-109 — 플립 구조의 정확한 수치: 스테이지 perspective:1200px, 래퍼 transform-style:preserve-3d; transition:transform 0.55s cubic-bezier(0.3,0.7,0.2,1), 공개 시 rotateX(-180deg), 앞·뒷면 position:absolute; inset:0; backface-visibility:hidden(뒷면은 rotateX(180deg) 선회전). 하단 판정 영역 min-height:132px; align-items:flex-end; padding-top:18px, 버튼 아래 12px 라벨, 미공개 시 힌트 문구 margin:0 0 34px.

3설계

  • 컴포넌트 구조: Mode1Card를 플립 구조로 재작성 — .flip-stage(flex:1, perspective, min-height:0) 안에 탭 대상인 button.flip-card(preserve-3d 래퍼, revealed면 .flip-card--revealed로 rotateX(-180deg)) > 앞면/뒷면 .flip-face 두 장. 공개 후엔 disabled로 재탭 무력화(플레이스홀더와 동일 패턴).
  • 한자 크기 스케일: 순수 헬퍼 hanziSizeClass(hanzi)를 src/lib/hanziSize.ts로 분리해 vitest(node)로 검증 — 이 레포는 컴포넌트 테스트 환경(jsdom/testing-library)이 없어 로직을 lib로 빼는 게 기존 패턴(wordState·sessionQueue와 동일 배치). 글자 수는 Array.from(hanzi).length(코드포인트 기준): 2 이하 84px / 3 = 64px / 4 이상 52px. CSS 클래스 3종(--s2/--s3/--s4)으로 매핑.
  • 뒷면 텍스트: 공유 클래스 .mode-card-hanzi(44px)·.mode-card-pinyin·.mode-card-meaning을 재사용 — Mode2Card(#17)와 수치가 동일 사양이므로 중복 정의하지 않는다.
  • O/X 버튼: .judge를 §4.2 원형 사양으로 재정의(82px 원형, 글리프 32px/800, active scale 0.94 유지, 색 틴트는 기존 .judge--x/--o 유지). .judge는 Mode1 전용이라 안전. 버튼+라벨("몰랐음"/"알고 있었음")을 세로 묶음으로, 등장은 래퍼에 fadeup 0.3s.
  • 기각한 대안: ① 플립을 keyframe 애니메이션으로 — 역방향 재생·중간 상태 관리가 불필요하게 복잡, 목업도 transition. ② 크기 스케일을 인라인 style로 — 토큰·수치를 CSS에 모으는 기존 컨벤션(§1 토큰 주석) 위배. ③ 컴포넌트 렌더 테스트 추가 — jsdom 셋업 도입은 이 이슈 범위 밖, 로직은 lib 헬퍼로 커버.

4작업 단계

  1. ✅ 한자 크기 스케일 순수 헬퍼 + 단위 테스트 — 코드포인트 수 기준 84/64/52px 결정 (2자 이하·3자·4자 이상·서로게이트 쌍 케이스) src/lib/hanziSize.ts, src/lib/hanziSize.test.ts
  2. ✅ Mode1Card를 3D 플립 구조로 재작성 — 앞면(한자 크기 스케일 + "탭해서 뜻 보기"), 뒷면(한자 44px + 병음 + 뜻), 탭 → 플립, 공개 후 X("몰랐음")·O("알고 있었음") 원형 버튼 + 미공개 힌트. onJudged 즉시 호출 계약 유지, 모든 한자 요소 lang="zh-Hans" src/screens/Mode1Card.tsx
  3. ✅ CSS: 플립 스테이지·카드면·원형 judge 버튼·판정 영역 스타일 — 목업 수치 그대로(perspective 1200 / 0.55s cubic-bezier(0.3,0.7,0.2,1) / 뒷면 rotateX(180deg) / fadeup 0.3s). 플레이스홀더 전용이던 .judge 재정의, Mode2 공유 클래스는 불변. 노트: 판정 영역은 mode-area gap 14px을 차감해 min-height 118px + padding-top 4px로 목업의 132/18px 리듬을 유지. src/index.css
  4. ✅ 정적 검증 — npm run test 68개 통과(기존 64 + 신규 4), npm run lint·npm run build 클린 vitest.config.ts 기준, 코드 변경 없음
  5. ✅ 실기기 흐름 검증 — .dev.vars 복사(gitignored) 후 npm run dev(5174 — 5173은 메인 체크아웃의 다른 서버가 선점), 390px 컬럼으로 로그인→학습 시작→모드1에서 플립·O/X·즉시 진행 실확인. 노트 1: 기록 API는 페이지에서 스텁(500 반환)해 실제 구글 시트 오염 방지 — POST 발사 여부는 스텁 로그로 관측(O→POST /api/answer mode:m1 1건, X→0건). 노트 2: X 후 분모 증가는 큐가 이미 60=SESSION_CAP이라 이 데이터로는 관측 불가한 것이 정상 동작 — 재삽입 규칙은 #15의 studySession.test.ts가 커버. 노트 3: 모드2 회귀는 브라우저 연결 종료로 CSS diff 정적 확인으로 대체 — 모드2가 쓰는 클래스는 diff에 없음(불변). 브라우저 검증 (코드 변경 없음)
  6. ✅ 플랜 체크오프·커밋(레포 스타일 한국어 메시지)·push·PR 오픈(Closes #16) docs/plans/issue-16-quiz-mode1-ui.html

5리스크

리스크영향완화책
.judge 재정의가 다른 화면에 파급낮음 — 스타일 붕괴grep으로 사용처 확인 완료: Mode1Card.tsx에서만 사용. Mode2는 primary-button 사용.
플립 뒷면이 Safari(iOS)에서 비치는 backface 버그중간 — 앞면에 뒷면 텍스트 겹침목업과 동일하게 -webkit-backface-visibility:hidden 병기, 뒷면 rotateX(180deg) 선회전 방식 유지.
같은 단어의 재삽입 문제에서 플립 상태 잔존중간 — 뒷면부터 보이는 스포일러셸이 key={session.pos}로 리마운트(StudyScreen.tsx:132) — 상태 자동 초기화. 검증 단계에서 X→재출제 흐름으로 실확인.
플립 transition 중 O/X가 먼저 떠서 뒷면 스포일러 전 판정 가능낮음 — UX 어색함목업도 동일(공개 즉시 fadeup) — 사양대로 두되 검증 단계에서 체감 확인만.
